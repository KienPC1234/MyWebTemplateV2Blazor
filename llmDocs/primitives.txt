# NeoUI -- Primitives Reference

NeoUI.Blazor.Primitives provides 15 headless, unstyled components with full WCAG 2.1 AA
accessibility, keyboard navigation, and ARIA attributes. Use them to build a fully custom
design system without any opinionated styling.

## Installation

```bash
dotnet add package NeoUI.Blazor.Primitives --version 3.8.3
```

Note: Already included transitively when using `NeoUI.Blazor`.

## Import

```razor
@using NeoUI.Blazor.Primitives
@using NeoUI.Blazor.Primitives.Services
```

## Service Registration

```csharp
builder.Services.AddNeoUIPrimitives();
```

## Building on Primitives

Example: custom-styled Dialog using the headless primitive:

```razor
@using NeoUI.Blazor.Primitives.Dialog

<Dialog @bind-Open="isOpen">
 <DialogTrigger>
  <button class="my-custom-button">Open</button>
 </DialogTrigger>
 <DialogPortal>
  <DialogOverlay class="my-overlay" />
  <DialogContent class="my-dialog-content">
<h2>My Custom Dialog</h2>
<p>All behavior, focus management, and ARIA handled automatically.</p>
<DialogClose>
 <button class="my-close-button">×</button>
</DialogClose>
  </DialogContent>
 </DialogPortal>
</Dialog>
```

## Services

### IKeyboardShortcutService

```csharp
@inject IKeyboardShortcutService KeyboardShortcuts

await KeyboardShortcuts.RegisterAsync("Ctrl+K", OpenPalette);
await KeyboardShortcuts.UnregisterAsync("Ctrl+K");
```

## Sortable Primitives (v3.8.0+)

Headless drag-and-drop sortable components with the same behavior as the styled `Sortable<TItem>` family but with no opinionated CSS. Use these to build fully custom sortable UI.

### SortablePrimitive (Layout)

```razor
@using NeoUI.Blazor.Primitives.Sortable

<SortablePrimitive TItem="MyItem"
                   Items="@items"
                   GetItemId="i => i.Id"
                   OnItemsReordered="@(list => items = (List<MyItem>)list)"
                   Context="scope">
 <SortableContentPrimitive>
  @foreach (var item in items)
  {
   <SortableItemPrimitive Value="@item.Id" @key="item.Id">
    <SortableItemHandlePrimitive />
    @item.Name
   </SortableItemPrimitive>
  }
 </SortableContentPrimitive>
 <SortableOverlayPrimitive Context="activeId">
  <div>Dragging @activeId</div>
 </SortableOverlayPrimitive>
</SortablePrimitive>
```

**Sub-components:**

| Component | Key Parameters | Notes |
|-----------|---------------|-------|
| `SortablePrimitive<TItem>` | Same as styled `Sortable<TItem>` (Items, GetItemId, Orientation, OnItemsReordered, OnDragStart, OnDragEnd, OnDragCancel, Context, Group, OnItemTransferredIn, OnItemTransferredOut, OnCanDrop) | Headless container |
| `SortableContentPrimitive` | `Class`, `ChildContent` | Container div; gets `data-state="over"` on cross-list hover |
| `SortableItemPrimitive` | `Value` (required), `AsHandle`, `Class`, `ChildContent` | Item wrapper |
| `SortableItemHandlePrimitive` | `Class`, `ChildContent` | Grip handle button |
| `SortableOverlayPrimitive` | `Class`, `ChildContent` (`RenderFragment<string>?`) | Drag ghost portal |
